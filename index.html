README

Qu'est-ce que Datadog Preflight ?

Une vérification en une commande, qui teste depuis votre propre environnement l'accès aux services Datadog et vous dit en clair ce qui bloque, avant d'installer quoi que ce soit.

Ce que le contrôle vérifie

Depuis votre environnement, là où tournera l'agent :

  • Résolution DNS des services Datadog
  • Pare-feu : sortie sur le port 443
  • Détection d'un proxy
  • Endpoints APM, logs, métriques…

Liens

  • Code source sur GitHub
  • Documentation réseau Datadog

What is Datadog Preflight?

A one-command check that tests connectivity to Datadog services from your own environment and tells you clearly what's blocking, before you install anything.

What the check verifies

From your environment, where the agent will run:

  • DNS resolution of Datadog services
  • Firewall: outbound port 443
  • Proxy detection
  • APM, logs, metrics endpoints…

Guide d'installation — Kubernetes

Installation de l'agent Datadog sur Kubernetes avec Helm.

Quelques définitions :

  • Kubernetes (K8s) — système qui orchestre vos applications dans des conteneurs.
  • Helm — équivalent d'un "App Store" pour Kubernetes : installe des applications complexes en quelques commandes.
  • Agent Datadog — programme qui tourne sur votre cluster et envoie métriques, logs et traces vers Datadog.

Prérequis :

  • Un cluster Kubernetes fonctionnel
  • kubectl installé et configuré
  • helm installé
  • Une clé API Datadog — Organization Settings › API Keys
1 Ajouter le dépôt Helm Datadog

Helm fonctionne avec des dépôts (catalogues d'applications). On commence par enregistrer le dépôt officiel Datadog.

helm repo add datadog https://helm.datadoghq.com helm repo update
2 Créer un namespace dédié

Un namespace est un espace isolé dans Kubernetes. On en crée un dédié pour garder le cluster organisé.

kubectl create namespace datadog
3 Stocker la clé API de façon sécurisée

On ne met jamais une clé API directement dans un fichier de configuration. On utilise un Secret Kubernetes.

kubectl -n datadog create secret generic datadog-secret \ --from-literal api-key=VOTRE_CLE_API_ICI
Remplacez VOTRE_CLE_API_ICI par votre vraie clé API Datadog.
4 Créer le fichier de configuration

Créez un fichier datadog-values.yaml avec le contenu suivant. Le site est automatiquement adapté à votre sélection.

clusterName est un nom libre uniquement utilisé pour l'affichage dans Datadog, indépendant du nom réel du cluster. Pour activer les logs, décommentez le bloc logs.
5 Installer l'agent

On lance l'installation avec Helm en lui indiquant notre fichier de configuration.

helm install datadog-agent \ -f datadog-values.yaml \ datadog/datadog \ --namespace datadog
6 Vérifier l'installation

Quelques minutes après l'installation, vérifiez que les pods sont bien en statut Running.

Lister les pods

kubectl get pods -n datadog

Diagnostiquer un pod en erreur

kubectl describe pod <nom-du-pod> -n datadog
7 Vérifier dans l'interface Datadog

Rendez-vous sur app.datadoghq.com et naviguez vers Infrastructure › Kubernetes. Votre cluster devrait apparaître dans les minutes qui suivent.

Mettre à jour la configuration

Modifiez datadog-values.yaml puis lancez :

helm upgrade datadog-agent \ -f datadog-values.yaml \ datadog/datadog \ --namespace datadog
Désinstaller l'agent
helm uninstall datadog-agent --namespace datadog
Résolution des problèmes courants

Les pods ne démarrent pas

kubectl describe pod <nom-du-pod> -n datadog kubectl logs <nom-du-pod> -n datadog

Erreur "secret not found"

Vérifiez que le secret a bien été créé dans le bon namespace :

kubectl get secret datadog-secret -n datadog

Le cluster n'apparaît pas dans Datadog

  • Vérifiez que votre clé API est correcte
  • Vérifiez que les pods sont en statut Running
  • Attendez 5 minutes, la remontée des données peut prendre du temps

Installation guide — Kubernetes

Install the Datadog Agent on your Kubernetes cluster using Helm.

A few definitions:

  • Kubernetes (K8s) — a system that manages and orchestrates your applications in containers.
  • Helm — the "App Store" for Kubernetes: installs complex applications with a few commands.
  • Datadog Agent — a program running on your cluster that sends metrics, logs and traces to Datadog.

Prerequisites:

  • A working Kubernetes cluster
  • kubectl installed and configured
  • helm installed
  • A Datadog API key — Organization Settings › API Keys
1 Add the Datadog Helm repository

Helm uses repositories (application catalogs). Start by registering the official Datadog repository.

helm repo add datadog https://helm.datadoghq.com helm repo update
2 Create a dedicated namespace

A namespace is an isolated space in Kubernetes. Create one dedicated to Datadog to keep the cluster organized.

kubectl create namespace datadog
3 Store your API key securely

Never put an API key directly in a configuration file. Use a Kubernetes Secret instead.

kubectl -n datadog create secret generic datadog-secret \ --from-literal api-key=YOUR_API_KEY_HERE
Replace YOUR_API_KEY_HERE with your actual Datadog API key.
4 Create the configuration file

Create a file named datadog-values.yaml with the content below. The site is automatically adapted to your selection.

clusterName is a free-form name used only for display in Datadog, independent of the actual cluster name. To enable log collection, uncomment the logs block.
5 Install the agent

Run the installation with Helm, pointing it to our configuration file.

helm install datadog-agent \ -f datadog-values.yaml \ datadog/datadog \ --namespace datadog
6 Verify the installation

A few minutes after installation, check that pods are in Running status.

List pods

kubectl get pods -n datadog

Diagnose a failing pod

kubectl describe pod <pod-name> -n datadog
7 Check in the Datadog interface

Go to app.datadoghq.com and navigate to Infrastructure › Kubernetes. Your cluster should appear within a few minutes.

Update the configuration

Edit datadog-values.yaml then run:

helm upgrade datadog-agent \ -f datadog-values.yaml \ datadog/datadog \ --namespace datadog
Uninstall the agent
helm uninstall datadog-agent --namespace datadog
Troubleshooting

Pods are not starting

kubectl describe pod <pod-name> -n datadog kubectl logs <pod-name> -n datadog

"secret not found" error

Check that the secret was created in the correct namespace:

kubectl get secret datadog-secret -n datadog

Cluster doesn't appear in Datadog

  • Verify your API key is correct
  • Check that pods are in Running status
  • Wait 5 minutes — data ingestion can take some time
